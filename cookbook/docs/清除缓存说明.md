# 清除项目缓存说明

当项目代码逻辑更新（特别是涉及到核心计算逻辑如笔、线段的划分）时，为了确保计算结果的准确性，需要清除旧的计算缓存。

## 为什么要清除缓存

本项目会将计算好的缠论数据（分型、笔、线段等）缓存到本地文件中，以提高后续加载速度。
如果修改了核心计算逻辑（例如 `src/chanlun/cl.py` 中的代码），旧的缓存数据可能与新逻辑不符。如果不清除缓存，系统可能会继续使用旧的计算结果，导致修改不生效或出现数据异常。

## 如何清除缓存

### 方法一：使用内置脚本（推荐）

项目提供了一个专门用于清除缓存的脚本。在项目根目录下，运行以下命令：

```bash
python src/chanlun/clean_cache.py
```

该脚本会自动定位缓存目录并删除其中的所有缓存文件。

### 方法二：手动删除

如果需要手动删除，请找到项目数据目录下的 `cl_data` 文件夹并将其内容清空。

默认数据目录为：`项目根目录/.chanlun_pro`

即删除以下目录中的所有文件：
`D:\quantitative\chanlun-pro\.chanlun_pro\cl_data`

（注：具体路径可能根据操作系统和配置略有不同，建议使用方法一）

## 验证

清除缓存后，重新运行策略或 Web 服务，系统会自动根据最新的代码逻辑重新计算数据并生成新的缓存。
