### 🌳 缠论核心逻辑脉络（自底向上构建）

缠论是由“缠中说禅”提出的一套完整、自洽的市场结构分析理论，其核心思想是：**走势终完美**，即任何级别的走势最终都会完成其结构。以下是其由微观到宏观、由结构到策略的完整逻辑链条：

---

#### 1️⃣ **包含关系（包含处理）**
- **作用**：解决相邻K线价格重叠问题，使价格序列“干净”、可分析。
- **技术本质**：对相邻K线进行合并（向上包含取高高、低高；向下包含取低低、高低）。
- **量化关键**：需递归处理，直到无包含关系为止。
- **重要性**：是后续所有结构识别（分型、笔等）的前提。

---

#### 2️⃣ **分型（Fractal）**
- **顶分型**：某K线高点高于左右两根K线高点，且低点也高于左右（典型“M”结构）。
- **底分型**：某K线低点低于左右两根K线低点，且高点也低于左右（典型“W”结构）。
- **注意**：分型必须建立在**无包含关系**的K线上。
- **量化难点**：需判断分型有效性（如是否被后续K线破坏）。

---

#### 3️⃣ **笔（Bi / Stroke）**
- **定义**：连接相邻的顶分型与底分型，中间至少有一根独立K线（或满足最小K线数量，如5根）。
- **方向**：向上笔（底→顶），向下笔（顶→底）。
- **规则**（常见标准）：
  - 顶底分型间不能有包含；
  - 顶底分型之间至少间隔4根K线（即笔至少包含5根K线）；
  - 新笔必须破坏前一笔的高低点。
- **量化关键**：笔的确认具有**滞后性**，需处理“假突破”和“笔延伸”。

---

#### 4️⃣ **线段（Segment / Xianduan）**
- **定义**：由至少三笔构成，且方向一致（例如：上-下-上构成一个上涨线段）。
- **关键特征**：线段的破坏需**反向线段成立**（即必须形成反方向的至少三笔结构）。
- **分型笔线段递进**：分型 → 笔 → 线段，代表从微观到中观的走势单元。
- **量化难点**：线段划分存在“特征序列”方法（更严谨但复杂），需处理线段的**起始点修正**问题。

---

#### 5️⃣ **中枢（Zhongshu / Central Pivot / Trading Range）**
- **定义**：某级别走势中，**至少三个连续次级别走势**重叠的**价格区间**。
  - 对于日线级别，中枢由至少三个1分钟/5分钟/30分钟（依次级别定）的重叠区间构成。
  - 数学表达：中枢 Z = [max(三段低点), min(三段高点)]，且 max(低) < min(高)。
- **类型**：
  - 盘整中枢（一个中枢）
  - 趋势中枢（两个及以上同向不重叠中枢）
- **量化关键**：需递归识别次级别结构；中枢可扩张、延伸、新生。

---

#### 6️⃣ **走势类型（Trend vs. Ranging）**
- **盘整走势**：仅包含一个中枢。
- **趋势走势**：包含两个或以上**同方向、不重叠**的中枢。
  - 上涨趋势：中枢依次抬高；
  - 下跌趋势：中枢依次降低。
- **缠论核心**：任何走势必为“盘整”或“趋势”之一。

---

#### 7️⃣ **背驰（Divergence / Beichi）**
- **定义**：当前走势段（通常是线段或笔）在创新高/新低时，**力度减弱**（如MACD面积、黄白线高度、成交量等）。
- **分类**：
  - **趋势背驰**：发生在趋势最后一个中枢的前后段。
  - **盘整背驰**：发生在单一中枢内部的两个反向段。
- **量化指标**：
  - MACD柱状图面积（最常用）；
  - 斜率、动量、成交量衰减等。
- **注意**：背驰 ≠ 立即反转，需结合**买卖点**确认。

---

#### 8️⃣ **买卖点（Buy/Sell Points）**
缠论定义三类标准买卖点（全部基于**完全分类**）：

- **第一类买卖点（1st Buy/Sell Point）**：
  - 趋势背驰后的**极值点**（如下跌趋势背驰后的最低点）。
  - **最安全、最高效**的入场点。
- **第二类买卖点（2nd Buy/Sell Point）**：
  - 第一类买点后，**第一次回调不创新低**（或卖点后反弹不创新高）。
  - 确认第一类点有效，风险略高但机会更多。
- **第三类买卖点（3rd Buy/Sell Point）**：
  - 在中枢上方/下方形成**回抽不进入中枢**的点。
  - 用于确认趋势延续，适合加仓或突破追入。

> ✅ 所有买卖点都基于“走势终完美”和“完全分类”，**不测而测**。

---

#### 9️⃣ **多级别联立（Multi-Timeframe Analysis）**
- 缠论强调**级别联立**：大级别定方向，小级别找买卖点。
- 例如：日线为下跌趋势，30分钟出现第一类买点 → 可做短线反弹，但不逆势做多日线。
- 量化实现需构建**多级别结构树**（如日线→30分钟→5分钟）。

## 二、基础数据约定

### 2.1 原始输入
```python
class RawBar:
    timestamp: int  # Unix 毫秒时间戳
    open: float
    high: float
    low: float
    close: float
```
- 输入：`raw_bars: List[RawBar]`，时间升序，无缺失。

### 2.2 处理后K线（CleanBar）
```python
class CleanBar:
    # 逻辑价格（经包含处理）
    high: float
    low: float
    open: float
    close: float
    timestamp: int  # 通常取合并中**最后一根原始K线**的时间

    # 原始映射（关键！用于画线、回测、信号对齐）
    original_start_idx: int  # 合并所含原始K线起始索引
    original_end_idx: int    # 合并所含原始K线结束索引
    original_high: float     # 原始K线中的最高 high
    original_low: float      # 原始K线中的最低 low
```
> ✅ **所有结构均基于 `CleanBar` 序列计算，但信号时间/价格必须回溯至原始K线**。

## 三、包含处理（Merge Included Bars）

### 3.1 定义
两相邻K线满足：
- `(bar1.high ≥ bar2.high 且 bar1.low ≤ bar2.low)` 或
- `(bar1.high ≤ bar2.high 且 bar1.low ≥ bar2.low)`

### 3.2 合并方向判定
| 情况 | 方向判定依据 |
|------|------------|
| 已有确认笔 | 取**最后一笔的 direction**（1=up, -1=down） |
| 无笔但 ≥2 CleanBar | 用 `clean[-2].close` vs `clean[-1].close` |
| 初始两根 | 用当前两根的 close 比较 |

### 3.3 合并规则
| 趋势方向 | 合并后 high | 合并后 low |
|----------|-------------|------------|
| 向上     | max(high1, high2) | max(low1, low2) |
| 向下     | min(high1, high2) | min(low1, low2) |

> 🔸 “高高、低高” = 向上合并取高点中的高、低点中的高；
> 🔸 “低低、高低” = 向下合并取高点中的低、低点中的低。

### 3.4 算法流程（单向遍历，无全局回溯）
```python
clean = [create_clean_bar(raw_bars[0])]
direction = 0

for i in range(1, len(raw_bars)):
    current = create_clean_bar(raw_bars[i])
    last = clean[-1]
    
    if is_included(last, current):
        # 确定方向（略，见上）
        merged = merge(last, current, direction)
        # 更新原始映射
        merged.original_start_idx = last.original_start_idx
        merged.original_end_idx = current.original_end_idx
        merged.original_high = max(last.original_high, current.original_high)
        merged.original_low = min(last.original_low, current.original_low)
        clean[-1] = merged  # 替换最后一根
    else:
        clean.append(current)
        direction = 1 if last.close < current.close else -1
```
> ✅ **关键澄清**：
> - 合并后**计为1根 CleanBar**；
> - **无需从头回溯**，因 Clean 序列每步保持“无包含”；
> - `timestamp` 取**最后一根原始K线时间**。

## 四、分型（Fractal）

### 4.1 定义（基于 CleanBar）
- **顶分型**：
  - `clean[i].high > clean[i-1].high 且 clean[i].high > clean[i+1].high`
  - **且** `clean[i].low > min(clean[i-1].low, clean[i+1].low)`
- **底分型**：
  - `clean[i].low < clean[i-1].low 且 clean[i].low < clean[i+1].low`
  - **且** `clean[i].high < max(clean[i-1].high, clean[i+1].high)`

> ✅ 必须基于**无包含的 CleanBar**。

### 4.2 确认与破坏
- **标记**：分型出现即标记为 `candidate`；
- **破坏**：
  - 顶分型：后续 CleanBar.high > 分型.high；
  - 底分型：后续 CleanBar.low < 分型.low；
- **确认**：未被破坏 + 后续出现反向分型。

### 4.3 数据结构
```python
class Fractal:
    idx: int                # clean_bars 中索引
    price: float            # 顶=high, 底=low
    is_top: bool
    confirmed: bool
    original_timestamp: int # = raw_bars[original_idx].timestamp
    original_idx: int       # 通常取 clean_bars[idx].original_start_idx
```

## 五、笔（Bi）— 仅用“新笔”标准

### 5.1 构成规则
- 由**底分型 + 后续顶分型**（向上笔）或**顶分型 + 后续底分型**（向下笔）构成；
- **最小长度**：两分型**中心索引差 ≥ 4** → 笔包含 ≥5 根 CleanBar；
  - 例：底分型中心 i=1，顶分型中心 j=5 → j−i=4 → 合法；
- **突破验证**：
  - 向上笔：顶分型高点 > 前一笔的顶高点；
  - 向下笔：底分型低点 < 前一笔的底低点。

### 5.2 确认机制
- 仅当**后一分型 confirmed=True 且满足长度+突破**，笔才 confirmed；
- 否则为 `potential_bi`（待定，不用于交易）。

### 5.3 时间与价格锚定
- **起点时间** = 底分型对应**原始K线时间**；
- **起点价格** = 分型的 `original_low`（底）或 `original_high`（顶）。

### 5.4 数据结构
```python
class Bi:
    start_fractal_idx: int
    end_fractal_idx: int
    direction: int          # 1=up, -1=down
    start_price: float      # = original_low / original_high
    end_price: float
    start_time: int         # 原始K线时间
    end_time: int
    confirmed: bool
```

> ✅ **澄清**：
> - “中间至少一根独立K线” → 指 CleanBar 中两分型**不重叠**；
> - **主流采用 ≥5 CleanBar 标准**（更稳健）；
> - **老笔已废弃**，仅用新笔。

## 六、线段（Segment）— 特征序列法（唯一推荐）

### 6.1 为何用特征序列？
避免直接用笔高低点导致的毛刺误判，通过**次级别结构**提高转折判断准确性。

### 6.2 构建步骤（以上涨线段结束为例）

#### Step 1: 提取“向下笔”低点
- 从当前上涨线段中，取所有 `direction = -1` 的笔；
- 提取每笔的 `end_price`（即低点）。

#### Step 2: 构造虚拟K线
```python
virtual_bars = [CleanBar(high=p, low=p, open=p, close=p, timestamp=t, ...)]
```

#### Step 3: 对虚拟K线做包含处理
- **方向**：因是上涨中的回调 → 视为**向下趋势** → 合并取 `min(high), min(low)`

#### Step 4: 在 clean_virtual_bars 上找**顶分型**
- 若存在顶分型 → 标记“线段可能结束”。

#### Step 5: 等待确认
- 需再出现**一根新的向下笔**（且 confirmed）→ 原上涨线段正式结束。

> 🔁 **对称处理**：下跌线段结束 → 提取“向上笔”高点 → 找**底分型**。

### 6.3 数据结构
```python
class Segment:
    start_bi_idx: int
    end_bi_idx: int
    direction: int      # 1=up, -1=down
    low: float          # 线段最低
    high: float         # 线段最高
    confirmed: bool
```

## 七、中枢（Zhongshu）

### 7.1 定义
- 某级别走势中，**至少三个连续次级别走势**（线段或笔）的**价格重叠区间**；
- 数学表达：  
  `ZG = min(seg1.high, seg2.high, seg3.high)`  
  `ZD = max(seg1.low, seg2.low, seg3.low)`  
  若 `ZG > ZD`，则中枢 = `[ZD, ZG]`

### 7.2 扩展与新生
- 若第4段与中枢重叠 → **扩展中枢**：  
  `ZG = max(ZG, seg4.high)`, `ZD = min(ZD, seg4.low)`
- 若第4段不重叠 → **新生中枢**或**趋势形成**。

### 7.3 数据结构
```python
class Zhongshu:
    start_idx: int    # 起始线段索引
    end_idx: int      # 当前结束线段索引
    zg: float         # 中枢高
    zd: float         # 中枢低
    direction: int    # 由线段方向决定
```

## 八、走势类型

- **盘整走势**：仅包含**一个中枢**；
- **趋势走势**：包含**两个及以上同向、不重叠中枢**；
  - 上涨趋势：中枢依次抬高；
  - 下跌趋势：中枢依次降低；
- **缠论核心**：任何走势必为盘整或趋势之一。

---

## 九、背驰（Divergence）

### 9.1 定义
当前走势段（笔或线段）**创新高/新低，但力度减弱**。

### 9.2 力度量化（推荐）
- **MACD柱状图面积**（绝对值之和）；
- 或 `(high - low) × volume`、斜率衰减等。

### 9.3 程序化策略
- **不等极值点**，而是实时计算；
- 新段创新低但力度 < 前段 → `potential_beichi = True`；
- **确认** = 出现**反向笔 confirmed**。

> ✅ **背驰 ≠ 立即反转**，需结合买卖点确认。

## 十、三类买卖点（程序化实现）

| 买卖点 | 触发条件 | 确认方式 | 信号位置 |
|--------|--------|--------|--------|
| **第一类** | 趋势背驰 + 创新低/高 | **反向笔 confirmed** | 反向笔起点（即底/顶分型） |
| **第二类** | 1st后，回调不破前低（买）/不破前高（卖） | 价格**反弹突破回调高点** | 回调结束点 |
| **第三类** | 中枢形成后，价格回抽**未进入中枢** | 价格**再度上涨突破回抽高点** | 再度上涨起点 |

### 10.1 第三类买卖点（3买）详解
- **条件**：价格从上方回落，**最低点 > 中枢.ZG**；
- **确认**：随后**上涨突破该回落的高点**；
- **避免假突破**：必须等“再度上涨”确认。

> ✅ 所有买卖点均**后置确认**，但可标记 `potential` 用于预警。

## 十一、多级别联立

- **原则**：大级别定方向，小级别找买卖点；
- **示例**：日线下跌趋势中，30分钟出现1买 → 可做**短线反弹**，但**不逆势做多日线**；
- **量化实现**：构建**多级别结构树**（日线 → 30分钟 → 5分钟）；
- **信号过滤**：仅交易与大级别方向一致的小级别信号。

## 十二、处理“当下性”与“虚拟结构”

| 概念 | 程序化处理 |
|------|----------|
| **当下性** | 任何信号仅用 `t ≤ now` 数据，**禁止未来函数** |
| **待定结构** | 标记 `confirmed=False`，用于预测但不交易 |
| **潜在信号** | 满足部分条件（如背驰+新低）→ `potential_signal` |
| **实盘策略** | **仅交易 `confirmed=True` 信号** |
| **回测分析** | 可统计 `potential` 信号的胜率、盈亏比 |

```
分型列表（含 confirmed）
   ↓ [新笔规则：长度≥5 + 突破前笔]
笔列表
   ↓ [特征序列法 + 反向笔确认]
线段列表
   ↓ [三线段重叠 + 动态扩展]
中枢
   ↓ [MACD面积 + 反向笔确认]
背驰 → 第一类买卖点
   ↓ [价格行为监控]
第二、三类买卖点
   ↓ [多级别方向过滤]
最终交易信号（confirmed）
```

## 十四、关键模糊点终极澄清

| 问题 | 答案 |
|------|------|
| **“高高”是什么**？ | 向上包含合并时，`high = max(h1,h2)`, `low = max(l1,l2)` |
| **合并后算几根K线**？ | **1根** CleanBar，时间取最后一根原始K线 |
| **笔画在哪根原始K线上**？ | 起点/终点时间 = **分型所在原始K线时间**（通过 `original_idx` 映射） |
| **分型间隔4根怎么算**？ | 两分型**中心索引差 ≥4** → 笔 ≥5 根 CleanBar |
| **老笔 vs 新笔**？ | **只用新笔**（≥5 CleanBar + 突破前笔），老笔已废弃 |
| **是否需原始K线映射**？ | **必须**！用于画线、回测、信号对齐 |
| **包含处理需回溯吗**？ | **否**！单向遍历，合并后仅与前一根可能包含（实际不会） |
| **线段特征序列如何做**？ | 提取反向笔 → 构造虚拟K线 → 包含处理 → 找分型 → 等新笔确认 |

## 十五、程序化核心原则总结

| 概念 | 程序化关键 |
|------|-----------|
| **K线合并** | 按趋势方向取高高/低低，结果为1根K线 |
| **分型** | 三根 CleanBar 构成，需后续未破坏才确认 |
| **笔** | 两分型+≥5 CleanBar+突破前笔，确认才成立 |
| **线段** | 用特征序列法，通过次级别笔的分型判断转折 |
| **中枢** | 三段重叠区间，动态扩展 |
| **背驰** | 力度（MACD面积）衰减 + 反向笔确认 |
| **买卖点** | 全部需**后置确认**，但可提前标记潜在信号 |
| **当下性** | 任何信号必须基于 `t ≤ now` 数据，禁止未来函数 |
| **原始映射** | **必须维护**，确保信号与真实市场对齐 |
